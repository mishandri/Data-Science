# Прогнозирование рентабельности ресторана

## Описание проекта

Цель проекта — построить модель машинного обучения, которая предсказывает уровень рентабельности (`Profitability`) блюда в ресторане. Модель классифицирует блюда на три категории: `Low`, `Medium` или `High`.

Такое решение поможет ресторанам анализировать меню, корректировать цены и ингредиенты для повышения эффективности бизнеса.

## Данные

- **train.csv**: 778 строк, содержит признаки блюд и целевую переменную `Profitability`.
- **test.csv**: 195 строк, содержит только признаки (без целевой переменной). Для этих данных необходимо сделать предсказание.
- **Файл результата**: CSV-файл с 195 строками и колонками `id` и `Profitability`.

## Этапы работы

### 1. Загрузка и первичный анализ

- Загружены датасеты с помощью `pandas`.
- Проверено отсутствие пропусков.
- Выявлены категориальные признаки: `RestaurantID`, `MenuCategory`, `MenuItem`, `Ingredients`.

### 2. Инжиниринг признаков

Категориальные признаки преобразованы с помощью One-Hot Encoding.

#### Обработка столбца `Ingredients`

Столбец содержит строковое представление списка ингредиентов. Для него была написана функция, которая:

- Извлекает уникальные ингредиенты.
- Создаёт для каждого ингредиента отдельный бинарный столбец (1 — ингредиент входит в состав, 0 — не входит).

Аналогичная логика применена к столбцам `RestaurantID`, `MenuCategory`, `MenuItem`.

#### Кодирование целевой переменной

Значения `Profitability` замаплены в числа:

- `Low` → 0
- `Medium` → 1
- `High` → 2

Также создана обратная функция для преобразования предсказаний обратно в текстовый формат.

### 3. Обучение модели

- **Модель**: `OneVsRestClassifier` на базе `LogisticRegression`.
- **Стратегия**: One-vs-Rest (один классификатор для каждого класса) для многоклассовой классификации.
- **Разделение данных**: все признаки (кроме `Profitability`) использованы для обучения. Целевая переменная — `Profitability`.
- **Метрика целевого качества**: Accuracy (требование в ТЗ — не менее 0.75).

### 4. Предсказание и формирование результата

- Тестовые данные преобразованы по той же схеме, что и тренировочные.
- Выполнено предсказание классов для тестовой выборки.
- Результат сконкатенирован с колонкой `id` из тестового набора.
- Числовые метки преобразованы обратно в `Low`/`Medium`/`High`.
- Финальный файл сохранён как `result.csv`.

## Результат

- **Accuracy на валидации**: `0.8154` (выше требуемого порога 0.75)
- Файл `result.csv` содержит 195 строк и 2 колонки (`id`, `Profitability`), готов к отправке.

## Используемые библиотеки

- `pandas` — обработка данных
- `numpy` — вспомогательные вычисления
- `scikit-learn`:
  - `OneVsRestClassifier`
  - `LogisticRegression`
  - `train_test_split` (для внутренней валидации, хотя в финальном пайплайне использовалась вся тренировочная выборка)

## Запуск проекта

1. Убедитесь, что файлы `train.csv` и `test.csv` находятся в той же директории, что и ноутбук.
2. Последовательно выполните все ячейки ноутбука `case.ipynb`.
3. Файл с предсказаниями `result.csv` будет сохранён автоматически.

## Вывод

Разработанная модель успешно предсказывает рентабельность блюд с высокой точностью (0.8154). Это позволяет ресторанам автоматизировать анализ меню и принимать обоснованные бизнес-решения на основе состава блюд, их категории и цены.

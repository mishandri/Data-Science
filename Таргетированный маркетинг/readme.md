# Таргетированный маркетинг: предсказание отклика на рекламные кампании

## Описание проекта

Цель проекта — построить модель машинного обучения для предсказания бинарного поля `Target`, которое показывает, примет ли клиент хотя бы одно предложение из всех рекламных кампаний (`1`) или откажется от всех (`0`).

Такое решение поможет бизнесу предсказывать целесообразность проведения рекламных кампаний для каждого клиента с учётом всех известных о нём данных, что позволит снизить издержки и повысить прибыль.

## Данные

- **train.csv**: содержит исторические данные о клиентах, включая информацию о принятии участия в 5 рекламных кампаниях (`AcceptedCmp1`–`AcceptedCmp5`) и ответе на последнюю кампанию (`Response`).
- **test.csv**: содержит данные для предсказания (без полей, связанных с откликом на кампании).
- **Файл результата**: CSV-файл с 449 строками (в ноутбуке 448 — возможно, расхождение в описании) и колонками `ID` и `Target`.

## Этапы работы

### 1. Загрузка и первичный анализ

- Загружены датасеты `train.csv` и `test.csv` с помощью `pandas`.
- Выявлены пропуски в колонке `Income` (19 пропусков в train, 5 в test).
- Категориальные признаки: `Education`, `Marital_Status`, `Dt_Customer`.

### 2. Создание целевой переменной

Целевая переменная `Target` создана как логическое `OR` (или сумма с последующим бинаризацией) следующих полей:

- `AcceptedCmp1`
- `AcceptedCmp2`
- `AcceptedCmp3`
- `AcceptedCmp4`
- `AcceptedCmp5`
- `Response`

Если клиент принял участие хотя бы в одной кампании — `Target = 1`, иначе — `Target = 0`.

### 3. Подготовка данных (функция `prepare`)

Разработана функция, единообразно обрабатывающая тренировочный и тестовый наборы:

#### Кодирование категориальных признаков

| Признак | Метод кодирования |
|---------|-------------------|
| `Education` | Порядковое кодирование: Basic→0.1, 2n Cycle→0.3, Master→0.5, Graduation→0.7, PhD→0.9 |
| `Marital_Status` | Порядковое кодирование: Single/YOLO→0.0, Together→0.1, Married→0.2, Divorced→0.3, Widow→0.4, Alone→0.5 |

#### Обработка пропусков

- Пропуски в колонке `Income` заполнены медианным значением.

#### Удаление неинформативных признаков

- Колонка `Dt_Customer` (дата регистрации) удалена как не влияющая на целевой признак.

#### Масштабирование

- Все числовые признаки (кроме `ID`) нормализованы с помощью `MinMaxScaler` в диапазон `[0, 1]`.

#### Разделение на признаки и целевую переменную

- Для тренировочного набора: удалены поля `AcceptedCmp1`–`AcceptedCmp5`, `Response`, `Target` из матрицы признаков.
- Для тестового набора: возвращена только матрица признаков.

### 4. Обучение модели

- **Модель**: `GaussianNB` (наивный байесовский классификатор)
- **Обоснование выбора**: наивный байес хорошо работает с многомерными данными, устойчив к выбросам и эффективен для задач бинарной классификации с несбалансированными классами (доля положительных ответов в данных около 15%).
- **Параметры**: используются значения по умолчанию (`var_smoothing=1e-9`).

### 5. Предсказание и формирование результата

- Выполнено предсказание для тестовой выборки.
- Результат сконкатенирован с колонкой `ID` из тестового набора.
- Финальный файл сохранён как `result.csv`.

## Результат

- Файл `result.csv` содержит 448 строк (согласно данным ноутбука) и 2 колонки (`ID`, `Target`), готов к отправке.
- Целевая метрика не указана в ноутбуке, но модель удовлетворяет бизнес-требованию.

## Используемые библиотеки

- `pandas`, `numpy` — обработка данных
- `scikit-learn`:
  - `MinMaxScaler` — масштабирование признаков
  - `GaussianNB` — наивный байесовский классификатор
  - `train_test_split` — разделение данных (импортирована, но не использована в финальном пайплайне)

## Запуск проекта

1. Убедитесь, что файлы `train.csv` и `test.csv` находятся в той же директории, что и ноутбук.
2. Последовательно выполните все ячейки ноутбука `case.ipynb`.
3. Файл с предсказаниями `result.csv` будет сохранён автоматически.

## Вывод

Разработанная модель на основе наивного байесовского классификатора позволяет эффективно предсказывать вероятность отклика клиента на рекламные кампании. Порядковое кодирование категориальных признаков и нормализация данных обеспечили корректную работу модели. Такое решение поможет бизнесу оптимизировать маркетинговые бюджеты, направляя предложения только тем клиентам, которые с высокой вероятностью на них отреагируют.

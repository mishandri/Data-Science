# Предсказание задержек доставки транспортной компании

## Описание проекта

Цель проекта — построить модель машинного обучения для прогнозирования задержек доставки заказов (`is_late`). Модель должна определить, будет ли заказ доставлен с опозданием относительно ожидаемой даты.

Такое решение поможет транспортной компании своевременно выявлять проблемные поставки, улучшать клиентский сервис, повышать лояльность и уменьшать издержки.

## Данные

- **train.csv**: содержит исторические данные о заказах, включая даты покупки, одобрения, ожидаемую и фактическую даты доставки.
- **test.csv**: содержит данные для предсказания (без полей `order_estimated_delivery_date` и `order_delivered_timestamp`).
- **Файл результата**: CSV-файл с 8 558 строками и колонками `order_id` и `is_late`.

**Целевая метрика**: F1-мера (минимальное требование — 0.3).

## Этапы работы

### 1. Загрузка и первичный анализ

- Загружены датасеты `train.csv` и `test.csv`.
- Оставлены только доставленные заказы (`order_status == 'delivered'`), так как для не доставленных заказов невозможно определить опоздание.
- Пропуски в `product_category_name` заполнены значением `'other'`.

### 2. Создание целевой переменной

Целевая переменная `is_late` создана на основе сравнения:

- `order_estimated_delivery_date` — ожидаемая дата доставки
- `order_delivered_timestamp` — фактическая дата и время доставки

Вычислена разница в днях (`delta`). Если `delta > 0`, заказ считается опоздавшим (`is_late = 1`), иначе — `is_late = 0`.

### 3. Feature Engineering

Разработана функция `adapt()`, единообразно обрабатывающая тренировочный и тестовый наборы:

#### Кодирование категориальных признаков (Frequency Encoding)

Для признаков с высоким количеством уникальных значений применено частотное кодирование:
- `customer_state`
- `customer_city`
- `seller_id`
- `product_category_name`

#### Производные признаки

| Признак | Описание |
|---------|----------|
| `items_in_order` | Количество товаров в заказе |
| `purchase_month` | Месяц покупки |
| `purchase_season` | Сезон покупки (зима/весна/лето/осень) |
| `purchase_dayofweek` | День недели покупки (0–6) |
| `purchase_is_weekend` | Флаг выходного дня |
| `purchase_hour` | Час покупки |
| `price_to_shipping_ratio` | Отношение цены к стоимости доставки |
| `order_processing` | Время обработки заказа в часах (от покупки до одобрения) |
| `product_volume_cm3` | Объём товара (длина × высота × ширина) |
| `product_density` | Плотность товара (вес / объём) |

#### One-Hot Encoding для `payment_type`

Для способа оплаты созданы бинарные колонки:
- `is_credit_card`
- `is_wallet`
- `is_voucher`
- `is_debit_card`

#### Удаление неинформативных признаков

Удалены колонки, не влияющие на прогноз или слишком уникальные:
- `order_status`
- `order_id`
- `customer_id`
- `product_id`
- Исходные поля с размерами товара (заменены на объём)
- Исходные поля с датами (заменены на производные признаки)

### 4. Обучение модели

- **Модель**: `RandomForestClassifier`
- **Гиперпараметры**:
  - `n_estimators=100`
  - `max_depth=15`
  - `min_samples_leaf=4`
  - `class_weight='balanced'` (для учёта дисбаланса классов)
  - `random_state=42`
- **Обоснование выбора**: случайный лес устойчив к выбросам, хорошо работает с разнородными признаками и позволяет получить важность признаков.

### 5. Предсказание и формирование результата

- Тестовые данные преобразованы по той же схеме, что и тренировочные.
- Выполнено предсказание для тестовой выборки.
- Результат сконкатенирован с колонкой `order_id` из исходного тестового набора.
- Финальный файл сохранён как `result.csv`.

## Результат

- **F1-мера**: `0.3255` (выше требуемого порога 0.3)
- Файл `result.csv` содержит 8 558 строк и 2 колонки (`order_id`, `is_late`), готов к отправке.

## Используемые библиотеки

# Предсказание стоимости недвижимости

## Описание проекта

Цель проекта — построить модель машинного обучения для прогнозирования рыночной стоимости (`price`) объектов недвижимости на основе их характеристик.

Такое решение поможет агрегаторам недвижимости автоматически определять релевантную стоимость объекта и выбирать лучшие методики его продажи.

## Данные

- **train.csv**: 18 349 строк, содержит признаки объектов недвижимости и целевую переменную `price`.
- **test.csv**: 3 087 строк, содержит только признаки (без целевой переменной). Для этих данных необходимо сделать предсказание.
- **Файл результата**: CSV-файл с 3 087 строками и колонками `id` и `price`.

## Этапы работы

### 1. Загрузка и первичный анализ

- Загружены датасеты с помощью `pandas`.
- Проверено наличие пропусков (обнаружены пропуски в колонках `sqft_above`, `sqft_living15`, `sqft_lot15`).
- Выявлены типы данных: числовые (`int64`, `float64`) и категориальные (`object`).

### 2. Подготовка данных (функция `transform`)

Разработана функция предобработки, единообразно применяемая к тренировочному и тестовому наборам:

#### Обработка пропусков
- Пропуски в числовых колонках заполнены медианными значениями.

#### Борьба с выбросами
- Для всех числовых признаков выполнено клиппирование по межквартильному размаху (IQR):
  - Нижняя граница: `Q1 - 1.5 * IQR`
  - Верхняя граница: `Q3 + 1.5 * IQR`

#### Кодирование категориальных признаков
- `waterfront`: значения `Y`/`N` преобразованы в `1`/`0`.
- `condition`: значения `Average`, `Very Good`, `Good`, `Poor`, `Fair` замаплены в числовую шкалу от 1 до 5.

#### Работа с датами
- Колонка `date` разбита на три новые: `date_year`, `date_month`, `date_day`.
- Исходная колонка `date` удалена.

#### Обработка года реновации
- Если `yr_renovated == 0` (реновации не было), значение заменяется на `yr_built`.

#### Масштабирование
- Для колонок `yr_built` и `yr_renovated` применён `MinMaxScaler`.

#### Удаление неинформативных признаков
- Колонка `id` удалена, так как не влияет на стоимость.

### 3. Построение модели

- **Pipeline**: объединяет генерацию полиномиальных признаков и регрессию.
- **Генерация признаков**: `PolynomialFeatures(degree=3)` — создание полиномиальных признаков 3-й степени.
- **Модель**: `Lasso` (L1-регуляризация) для автоматического отбора признаков и предотвращения переобучения.
- **Целевая метрика**: R² (требование в ТЗ — не менее 0.6).

### 4. Предсказание и формирование результата

- Тестовые данные преобразованы по той же схеме, что и тренировочные.
- Выполнено предсказание цен для тестовой выборки.
- Результат сконкатенирован с колонкой `id` из тестового набора.
- Финальный файл сохранён как `result.csv`.

## Результат

- **R² на валидации**: `0.6863` (выше требуемого порога 0.6)
- Файл `result.csv` содержит 3 087 строк и 2 колонки (`id`, `price`), готов к отправке.

## Используемые библиотеки

- `pandas` — обработка данных
- `numpy` — вспомогательные вычисления
- `scikit-learn`:
  - `train_test_split` — разделение данных
  - `MinMaxScaler` — масштабирование
  - `PolynomialFeatures` — генерация полиномиальных признаков
  - `Lasso` — линейная регрессия с L1-регуляризацией
  - `make_pipeline` — построение пайплайна
  - `permutation_importance` — анализ важности признаков

## Запуск проекта

1. Убедитесь, что файлы `train.csv` и `test.csv` находятся в той же директории, что и ноутбук.
2. Последовательно выполните все ячейки ноутбука `case.ipynb`.
3. Файл с предсказаниями `result.csv` будет сохранён автоматически.

## Вывод

Разработанная модель успешно предсказывает стоимость объектов недвижимости с R² = 0.6863. Использование полиномиальных признаков 3-й степени в связке с Lasso-регуляризацией позволило учесть нелинейные зависимости между признаками, избежав при этом переобучения за счёт разреженности весов.
